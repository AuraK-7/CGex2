# 计算机图形学实验二

## 项目介绍

本项目是《计算机图形学》的实验二，目的是构建了一个基于 C++ 与 OpenGL 核心模式的轻量级 3D 实时渲染引擎。

按照实验要求，本项目 **不使用** Unity、Unreal 等任何成熟商业引擎。

项目以一个3D 教室为核心场景，重点实现了基于物理学公式的光照阴影算法、复杂的模型与材质加载系统、以及带有包围盒（AABB）碰撞检测的第一人称漫游控制。

## 技术选型与架构

为了保证开发的高效性与渲染的稳定性，我们选用了业界标准的跨平台图形学基础库：

| 模块类别 | 库名称 | 选型说明 |
| --- | --- | --- |
| **构建系统** | **CMake** | 跨平台标准构建工具，解耦特定 IDE 配置，确保团队成员可以在 VS、CLion 或 VS Code 环境下无缝切换。 |
| **窗口与上下文** | **GLFW** | 轻量级的窗口管理与事件回调库，处理跨平台窗口创建及精准的键盘、鼠标输入监听。 |
| **函数指针加载** | **GLAD** | 运行时加载 OpenGL 核心模式函数指针，抛弃老旧的固定管线 API。 |
| **核心数学库** | **GLM** | 专为图形编程设计的数学库，提供与 GLSL 规范高度一致的向量与矩阵运算（平移、旋转、缩放、投影）。 |
| **3D 模型解析** | **Assimp** | 业界顶级的资产导入库。支持解析复杂的 `.fbx` 格式，高效提取顶点、法线、UV 及层级结构数据。 |
| **图像数据解码** | **stb_image.h** | 极简的单头文件图像库，用于将外部 `.png` 等纹理文件读取为原始的 RGB/RGBA 像素数组。 |
| **即时渲染 UI** | **Dear ImGui** | 极其轻量、对图形 API 友好的 UI 库，用于快速构建实时参数调试面板。 |


## 编译与运行

### 1. 环境依赖

* **编译器**：支持 C++17 标准的编译器 (MSVC / GCC / Clang)
* **构建工具**：CMake (建议版本 3.15+)
* **资源说明**：受限于 GitHub 仓库容量管理，本项目的大型 3D 模型与贴图未纳入版本控制。

### 2. 资源配置

```text
ClassroomRenderer/
├── src/
├── shaders/
├── Thirdparty/           <-- 请自行去对应官网或仓库下载
|   ├── glad/           
|   ├── glfw/        
|   └── glm/         
└── resources/            <-- 必须手动放置于此
    ├── models/           # 包含 model.fbx
    └── textures/         # 包含所有 .png 贴图文件

```

### 3. 构建步骤

通过 CMake 生成构建文件并编译：

```bash
mkdir build
cd build
cmake ..
cmake --build . --config Release

```

### 🛠️ 全局核心技术选型清单 (C++ / OpenGL 栈)

在开始分配任务前，请全组统一以下第三方库的选型，并在 CMakeLists.txt 中配置好，这是你们的“武器库”：

| 模块类别 | 推荐库/技术 | 为什么选它？ |
| --- | --- | --- |
| **构建系统** | **CMake** | C++ 跨平台标准，方便所有人一键生成 VS 或 CLion 工程。 |
| **窗口与输入** | **GLFW** | 轻量级，完美处理键盘、鼠标回调以及跨平台窗口创建。 |
| **API 函数加载** | **GLAD** (或 GLEW) | OpenGL 核心模式必备，负责获取显卡驱动中的函数指针。 |
| **数学运算** | **GLM** (OpenGL Math) | 极其重要！提供了和 GLSL 着色器语言几乎一致的向量和矩阵类（如 `glm::mat4`）。 |
| **模型加载** | **Assimp** | 业界标准。能极其稳定地解析 `.fbx`，并保留层级和材质引用。 |
| **图片加载** | **stb_image.h** | 仅一个头文件，直接解析 PNG/JPG 为像素数组。 |
| **GUI 面板** | **Dear ImGui** | 游戏引擎最爱用的即时渲染 UI 库，无缝接入 OpenGL。 |

### 🗺️ 任务依赖与并行阶段全景图

图形学项目的最大特点是“必须先画出个丑东西，才能慢慢把它变好看”。以下是详细的任务清单及并行策略：

#### 阶段一：基础设施搭建 (绝对阻塞点，必须串行)

**【依赖关系】**：所有后续工作的前置条件。
在这个阶段，所有人都不需要碰复杂的模型，只需要在屏幕上画出一个**纯色的正方体**即可。

* **Task 1.1：工程初始化与黑屏渲染**
* **内容**：搭好 CMake，跑通 GLFW，能弹出一个黑色的 OpenGL 窗口。


* **Task 1.2：基础着色器类封装 (Shader Class)**
* **内容**：写一个 C++ 类，能读取硬盘上的 `.vert` 和 `.frag` 文本文件并编译链接成 OpenGL 的 Program。


* **Task 1.3：摄像机与矩阵系统 (MVP Matrices)**
* **内容**：引入 GLM，实现第一人称 FPS 漫游（WASD 移动，鼠标转动视角）。能够正确计算并下发 `View` 和 `Projection` 矩阵给着色器。



#### 阶段二：核心模块开发 (高度可并行)

**【依赖关系】**：一旦屏幕上能自由观看一个正方体，团队就可以**兵分三路**，完全并行开发！

* **📍 分支 A：资产管线 (Asset Pipeline)**
* **Task 2.1：模型解析 (Model & Mesh Class)**
* **内容**：接入 Assimp，读取 `.fbx`，解析顶点、法线、UV 坐标存入 VAO/VBO。


* **Task 2.2：纹理加载系统 (Texture Loader)**
* **内容**：接入 `stb_image.h`，读取 PNG，生成 OpenGL 纹理对象并在绘制时绑定。


* **里程碑**：能够在场景中渲染出艺术家提供的真实教室模型（此时依然没有光影，是纯色的贴图）。


* **📍 分支 B：光照与材质 (Lighting & Materials)**
* **Task 2.3：基础光照模型 (Blinn-Phong)**
* **内容**：在片段着色器中用数学公式写出环境光、漫反射、高光。


* **Task 2.4：多光源架构**
* **内容**：实现方向光（太阳）、点光源（带距离衰减公式）、聚光灯（带边缘平滑过渡公式，这是**进阶任务**之一）。


* **里程碑**：用正方体或球体测试，能够看到不同光源打在物体上的明暗变化。


* **📍 分支 C：交互与系统交互 (UI & Logic)**
* **Task 2.5：GUI 接入**
* **内容**：配置 ImGui，在屏幕上画出面板，将灯光颜色、强度等变量暴露给 UI 滑块。


* **Task 2.6：AABB 碰撞检测引擎 (进阶任务)**
* **内容**：写一套数学逻辑，拦截 Task 1.3 中的相机移动坐标，判断是否与场景中物体的边界框相交。





#### 阶段三：硬核攻坚与集成 (部分阻塞)

**【依赖关系】**：分支 A 和 B 必须完成并合并代码。模型能显示，光照也正常了，才能进行下一步。

* **Task 3.1：阴影映射 (Shadow Mapping)**
* **难度系数**：⭐⭐⭐⭐⭐ (极易出 Bug)
* **内容**：这是一个极其底层的任务。需要手写创建帧缓冲对象（FBO）和深度纹理。编写一套专门生成深度图的 Shader（First pass），再修改 Task 2.3 的光照 Shader 引入阴影判定逻辑（Second pass）。
* **技术点**：深入理解正交投影矩阵（针对方向光阴影）、透视分割和深度偏移（Shadow Bias）。



#### 阶段四：打磨与进阶效果 (再次并行)

**【依赖关系】**：基础要求（有光、有影、能走动、有 UI）已全部实现。

* **Task 4.1：材质细节提升**
* **内容**：尝试解析并使用艺术家提供的法线贴图（Normal Mapping）和 Gamma 校正，让教室看起来更逼真。


* **Task 4.2：场景动态元素 (进阶任务)**
* **内容**：实现刚体动画。给场景里特定的模型（如风扇、挂钟）单独传入一个随时间 `glfwGetTime()` 变化的旋转矩阵（Model Matrix）。


* **Task 4.3：参数调优与场景组装**
* **内容**：使用 ImGui 面板，对照艺术家给的参考图，耐心地把光源位置、颜色、阴影软硬度调到最佳状态，并将满意的参数硬编码保存。

### 📊 推荐的 6 人指派方案 (结合上文的阶段图)

为了防止互相覆盖代码，最好按“模块”而非“阶段”来分人：

| 角色 / 组员 | 负责 Task 列表 | 备注 |
| --- | --- | --- |
| **Infra/架构师 (1人)** | Task 1.1, 1.2, 3.1(主攻阴影) | **抗压位**。负责底层搭建和最难的阴影算法。阴影算不出来，全组效果减半。 |
| **相机/物理逻辑 (1人)** | Task 1.3, 2.6, 4.2 | 专注于**线性代数与空间几何**。保证在教室里走起来丝滑且不穿墙，兼顾动态物体。 |
| **资产解析工程师 (1人)** | Task 2.1, 2.2 | 负责**与文件系统打交道**。要把复杂的数据结构整理成清爽的 C++ 类提供给其他人。 |
| **光影图形学研究 (1人)** | Task 2.3, 2.4, 4.1 | 专注于**写 GLSL 着色器**和数学公式。不需要管 C++ 怎么传数据，只要保证公式算出来的光最漂亮。 |
| **UI 与系统集成 (1人)** | Task 2.5, 4.3 | **产品经理位**。负责对接所有人。把底层参数接到 UI 上，负责最后的打光和截图。 |
